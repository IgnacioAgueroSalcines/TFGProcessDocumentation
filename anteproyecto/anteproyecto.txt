	{\Large Contexto del proyecto}\\
El proyecto presente está propuesto desde CIC. Esta es una empresa que se dedica al consulting informático,y que dentro de ella, se desarrolla LUCA.\\


 LUCA es un producto que abstrae al usuario de realizar consultas individuales a los diversos recursos para obtener datos(ya sean servicios, bases de datos, o similar) bajo un único lenguaje común. Este producto permite crear y gestionar consultas. De esta forma el usuario solo se comunica con LUCA para obtener cualquier tipo de información.\\

	{\Large Motivación del proyecto}\\

 Actualmente surge la necesidad de poder anidar los diversos procesos\footnote{Un proceso se explica como una entidad que por sí misma tiene la capacidad de realizar llamadas a servicio para recoger datos en función de unas variables de entrada y de salida.} independientes entre sí, así como la dificultad para organizarlos, interactuar y ejecutarlos.\\
 
 Esto significa que cuando el usuario desea realizar una consulta a una fuente, y utilizar dichos resultados para poner en marcha una nueva, este debe de realizar las consultas separadas y pasar los datos de entrada manualmente. De esta forma para realizar dicho ejemplo, los pasos a seguir por el usuario lo obligan a estar navegando por las diferentes funcionalidades de la aplicación para alcanzar su objetivo.\\
 
 	{\Large Solución propuesta}\\
	
	Este proyecto decide centrar al usuario en una sola ventana de diálogo desde la que se pueda acceder a las diferentes consultas ya configuradas para crear flujos de ejecución de las mismas para obtener los resultados deseados.\\
	
	Pretende crear un componente de procesos genérico, el cual permitirá crear diagramas donde el usuario sea capaz de enlazar los diferentes procesos entre sí, utilizando para ello variables de entrada y de salida, así como ejecutarlos y obtener información de ellos.\\
	
		\begin{center}
		\includegraphics[width=12cm]{images/ejemploDiagrama}\\
	\end{center}
	
	
	Entre las características que se le exigen es que sea lo mas configurable posible y estar preparado para posibles futuros incrementos.
	Además se pretende que dicho proyecto sea fácil de usar e intuitivo, es decir, altamente usable.\\
	
	Este componente será capaz de proveer una soporte gráfico (mediante una herramienta o motor gráfico llamado GO.JS) y lógico para llevar a cabo la funcionalidad final.\\
	
	
	\newpage
	{\Large Arquitectura y/o tecnologías del proyecto}\\
	
	El componente genérico se compone de dos soportes principales, uno mediante la herramienta javascript GO.JS, el cuál se encargara de visualizar gráficamente los datos (aunque también contiene una lógica interna propia a desarrollar), y otro soporte lógico encargado presentar el patrón MVP (modelo vista presentador), que interactuará con la herramienta gráfica.\\
	
	\begin{center}
		\includegraphics{images/mvp}\\
	\end{center}
	
	
	Este ensamblado de herramientas se realizará usando Vaadin, el cuál, mediante un fichero conector escrito en javascript, se encargará de realizar las comunicaciones entre la parte lógica y la parte gráfica.\\
